STEP 01 ターミナルを開く

まずは ターミナル を起動

「ターミナル」とは、文字でパソコンに命令を出すアプリです。OS によって開き方が違うので、お使いの環境を選んでください。

macOS の場合 — 標準で入っています

ターミナルアプリは Mac に最初から入っているので、ダウンロードは不要です。

  1. ⌘ + Space で Spotlight 検索を開く
  2. 「ターミナル」と入力して Enter
  3. 黒い画面が出れば準備OK
💡 もっと使いやすく 本格的に使うなら iTerm2 や Warp もおすすめ。最初は標準ターミナルで十分です。

Windows の場合 — Windows Terminal をインストール

Windows 11 には標準で入っていますが、Windows 10 ではストアからインストールします。

  1. Microsoft Store を開く
  2. 「Windows Terminal」を検索してインストール
  3. スタートメニューから「Windows Terminal」を起動
Windows Terminal をダウンロード ↗
⚠️ 確認ポイント Claude Code をインストールする際、PowerShell(プロンプトが PS C:\)と CMD(C:\)で使うコマンドが違います。Windows Terminal を起動すると通常 PowerShell になります。

Linux の場合 — 標準で入っています

多くのディストリビューションでは Ctrl + Alt + T で起動できます。

もし反応しない場合は、アプリ一覧から「Terminal」「端末」を探して起動してください。

02
STEP 02 インストール
›

Claude Code を インストール

ターミナルに以下のコマンドを 1行コピーして貼り付け、Enter を押すだけ。Node.js などの事前準備は不要です(ネイティブインストーラ)。

ネイティブインストーラ(推奨)

ターミナルにこのコマンドを貼り付けて実行してください。

curl -fsSL https://claude.ai/install.sh | bash

Homebrew が入っている方はこちら

brew install --cask claude-code
💡 完了の合図 コマンド終了後にターミナルを 一度閉じて開き直す と確実です(PATH を反映させるため)。

PowerShell の場合(推奨)

Windows Terminal で開いた PowerShell に貼り付け。

irm https://claude.ai/install.ps1 | iex

CMD(コマンドプロンプト)を使っている方

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
⚠️ よくあるエラー 「'irm' は内部コマンドまたは外部コマンド…として認識されていません」と出たら、それは CMD です。CMD 用のコマンドを使うか、PowerShell に切り替えてください。

ネイティブインストーラ(推奨)

curl -fsSL https://claude.ai/install.sh | bash

npm 経由でも可(Node.js 18 以上が必要)

npm install -g @anthropic-ai/claude-code

インストールできたか確認

新しいターミナルを開いて、次のコマンドを実行します。

claude --version
✅ こうなれば成功 2.1.x や 2.x.x のようなバージョン番号が表示されればインストール完了です。例:
claude version 2.1.126

うまく動かないときは健康診断コマンドが便利です。

claude doctor

↑ 緑色の ✓ がたくさん並んでいれば OK。赤い ✗ があれば STEP 09 トラブル対処 へ。

はじめての 起動とログイン

プロジェクトのフォルダ(または作業したいフォルダ)に移動してから起動するのが基本です。

1. 作業フォルダを作って入る

下のコマンドをそのままコピペすると、デスクトップに test-project という新しいフォルダができて、その中に入ります。

cd ~/Desktop mkdir test-project cd test-project

↑ 3行ありますが、まとめてコピー&ペーストでOK。mkdir はフォルダを作る、cd は中に入るコマンドです。

✅ こうなれば成功 画面のプロンプト(行頭の表示)に test-project が含まれていれば、フォルダの中に入っています。
例:~/Desktop/test-project $ や PS C:\Users\...\test-project>

2. Claude Code を起動

claude

初回起動時はブラウザが自動で開いて認証画面が表示されます。Claude のアカウントでログインして「Allow / 許可する」をクリック。

✅ こうなれば成功 ブラウザでの認証後、ターミナルに戻ると Claude Code のロゴ(オレンジ色の星型)が表示され、入力待ち状態になります。> のようなプロンプトが点滅していればOK。

↑ ブラウザが自動で開かない場合は、ターミナルに表示された URL をコピーしてブラウザに貼り付けてください(STEP 09 のエラー5 参照)。

3. 話しかけてみる

あとは普通の日本語で話しかければOK。最初は挨拶でも何でも大丈夫です。

こんにちは!まず自己紹介してください
✅ こうなれば成功 Claude が日本語で自己紹介してくれれば、すべて正常に動いています 🎉
ここまできたら STEP 06「初プロジェクト」に進みましょう。

STEP 06 初プロジェクト
›

はじめての プロジェクト を作ってみよう

百聞は一実行にしかず。Claude Code を使って、自分用のシンプルな自己紹介サイトを一緒に作ってみましょう。所要時間は10分です。

🎯 今回作るもの

名前・自己紹介・趣味が載った、おしゃれな1ページ。HTML だけで作るので、ファイルを保存してそのままブラウザで見られます。

① 作業フォルダを作る

デスクトップに新しいフォルダを作って、その中に入ります。ターミナルで以下を実行:

cd ~/Desktop mkdir my-first-site cd my-first-site

↑ mkdir はフォルダを作る、cd はそのフォルダに入るコマンドです。

② Claude Code を起動

claude

これで Claude が、いま開いた空っぽのフォルダの中で待機している状態になります。

③ 最初のプロンプトを送る

下の文をそのまま Claude にコピーして送信してください。

📝 プロンプト

index.html というファイルを作って。シンプルでおしゃれな自己紹介ページにして。名前・短い自己紹介・趣味リスト・連絡先のセクションを入れて。デザインは白ベースに1色アクセント、CSS は同じファイル内に書いて。

Claude が「こういうファイルを作ろうと思います」と内容を提示してきます。問題なければ Enter キー(または「y」と入力して Enter)で承認します。

④ ブラウザで開いてみる

Claude が index.html を作り終えたら、Finder(Mac)/エクスプローラー(Windows)で my-first-site フォルダを開いて、index.html を ダブルクリック。

ブラウザに、あなたの自己紹介ページが表示されます 🎉

⑤ アレンジする(ここからが楽しい)

そのまま Claude にこんなふうに話しかけてみてください。1つずつ試すのがおすすめ。

  • 背景をダークモードに変えて
  • SNS のリンクボタンを追加して(X と Instagram)
  • スクロールしたら要素がふわっと出るアニメーションを足して
  • スマホでも見やすくなるよう調整して

変更後はブラウザを 更新(F5 / ⌘+R) すれば反映されます。

💡 これでマスター ここまでできたら、Claude Code の基本は身についています。あとは「何を作ろうかな」を考えるだけ。Todo アプリ、メモアプリ、家計簿、ブログ、ポートフォリオ、何でも作れます。

STEP 07 CLAUDE.md
›

CLAUDE.md でレベルアップ

Claude Code を本気で使うなら、これは必須。「プロジェクトの取扱説明書」を Claude に渡すことで、毎回の応答精度が劇的に上がります。

📋 CLAUDE.md って何?

プロジェクトのルートに置く、Claude 専用のメモ書きです。Claude は起動時にこのファイルを自動で読み込んで、毎回の会話の前提知識として使います。

これがないと…
  • 毎回プロジェクトの説明をしないといけない
  • Claude が独自の流儀でコードを書いてしまう
  • 同じ間違いを繰り返す
あると…
  • 「いつものスタイル」で書いてくれる
  • 使ってほしくないライブラリを避けてくれる
  • プロジェクト固有のルールを守ってくれる

🚀 作り方は2通り

方法A:Claude に作ってもらう(おすすめ)

プロジェクトのルートで Claude Code を起動して、このコマンドを実行するだけ。

/init

Claude がフォルダの中身を読み解いて、適切な CLAUDE.md を自動生成 してくれます。あとから自由に編集して大丈夫。

方法B:手動で作る

プロジェクトのルートに CLAUDE.md という名前のファイルを作って、Markdown で書きます。次のテンプレを参考にしてください。

📝 テンプレート(コピペで使える)

このまま使ってもOK。自分のプロジェクトに合わせて編集してください。

# Project: My Portfolio Site ## 概要 個人用のポートフォリオサイト。HTML/CSS/JS のみで作る。 ## ファイル構成 - index.html — トップページ - style.css — 全体のスタイル - script.js — インタラクション ## 開発ルール - コメントは日本語で書く - インデントはスペース2つ - 関数名はキャメルケース(例:getUserName) - 外部 CDN は使わない(オフラインでも動くように) ## やってほしいこと - コードを書く前に、何を変えるか日本語で説明する - 大きな変更は確認してから実行する - アクセシビリティ(alt属性、コントラスト)を意識する ## やってほしくないこと - jQuery を使う - インラインstyle属性を多用する - 既存の機能を勝手に削除する

✨ よく使う書き方の例

  • 📍 言語の指定 「コメントは日本語で書いてください」
  • 📍 説明の癖をつけさせる 「ファイルを変更する前に、必ず変更内容を箇条書きで説明してください」
  • 📍 ライブラリの制約 「React は使わず、Vanilla JS のみで書いてください」
  • 📍 テストの実行 「コードを変更したら必ず npm test を実行してエラーがないか確認してください」
  • 📍 安全策 「.env ファイルや config/secrets/ の中身は絶対に読んだり変更したりしないでください」
💡 上手に育てるコツ 最初から完璧を目指さなくてOK。使いながら「これも書いておけば伝わるな」というルールが見えてきたら、その都度追記するのが正解です。チームで使う場合は CLAUDE.md を Git で共有すると、メンバー全員の Claude が同じルールで動いてくれます。

STEP 09 トラブル対処
›

うまくいかない時の 対処法

初心者がつまずきやすいポイントを、やさしく具体的に解説します。エラーが出ても焦らないで大丈夫。ほぼ全部、数分で直せます。

📖 そもそもエラーって何?

ターミナルに赤い文字や英語のメッセージが出たら、それが「エラー」です。怖がらないで大丈夫。パソコンが「ここがわからないから教えて」と言っているだけです。

大事なのは、エラーの中に 解決のヒントが必ず書いてある ということ。「command not found」と書いてあれば「コマンドが見つからない」、「permission denied」なら「許可がない」という具合です。

エラーが出たら最初にすること
  1. エラー文を マウスで選択してコピー(Mac: ⌘+C / Windows: Ctrl+Shift+C)
  2. 右下の 💬 ボタン から Claude を開いて、エラー文を貼り付けて質問するのが一番速いです
  3. または、下の一覧から似たエラーを探してみる
これだけ知っておけばOKな用語
  • ターミナル — 黒い画面のこと。文字で命令を出すアプリ
  • コマンド — ターミナルに打つ「命令」(例:claude)
  • PATH(パス) — パソコン内の「アプリ住所録」。ここに登録されてないと、コマンドを呼んでも見つけられない
  • 権限(permission) — 「このフォルダを触っていいよ」という許可。これがないと書き込めない

🩺 困ったら、まずこの3つを順番に

  1. ターミナルを完全に閉じて、開き直す
    → 9割の問題はこれで直ります。「タブを閉じる」じゃなく「ウィンドウごと閉じる」が大事。
  2. claude doctor を実行
    → 健康診断のように、問題を自動で見つけて教えてくれます。
  3. 右下の 💬 ボタンから Claude にきく
    → エラー文をそのまま貼り付ければ、解決法を日本語で教えてくれます。

1. claude コマンドが見つからない

command not found: claude 'claude' is not recognized as an internal or external command
💬 やさしく言うと

パソコンに「claude って呼んで」と言ったのに、「そんな子知らないよ」と返されている状態。インストールはできていても、パソコンがまだ claude の 居場所(PATH) を覚えていません。

🔧 直し方(順番にやってみる)
  1. 今のターミナルウィンドウを完全に閉じる
    Mac: ⌘+Q でアプリごと終了 / Windows: ウィンドウ右上の ✕ ボタン
  2. 少し待ってから、新しくターミナルを起動する
  3. 次のコマンドを実行:
claude --version
✅ 直ったか確認

バージョン番号(例:2.1.x)が表示されれば成功です。それでも同じエラーなら、もう一度インストールコマンドを実行してみてください。

2. irm が認識されない(Windows)

'irm' is not recognized as an internal or external command, operable program or batch file.
💬 やさしく言うと

Windows には「PowerShell」と「CMD(コマンドプロンプト)」の 2種類のターミナル があります。irm は PowerShell 専用のコマンド。今あなたが開いているのは CMD のほうなので、動きません。

🔍 自分がどっちを使ってるか確認
  • 画面に PS C:\Users\〜> と書いてある → PowerShell(これでOK)
  • 画面に C:\Users\〜> だけ → CMD(切り替えが必要)
🔧 直し方

方法A:PowerShell に切り替える(おすすめ)

  1. スタートメニューを開く
  2. 「PowerShell」と検索
  3. 「Windows PowerShell」をクリック
  4. もう一度 irm https://claude.ai/install.ps1 | iex を実行

方法B:CMD のままで進める

CMD 用のコマンドを使ってください。

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

3. && でエラーが出る(Windows)

The token '&&' is not a valid statement separator in this version.
💬 やさしく言うと

2番のちょうど逆。今度は PowerShell に CMD 用のコマンドを打ち込んでしまっています。

🔧 直し方

PowerShell 用のコマンドを使ってください。

irm https://claude.ai/install.ps1 | iex
✅ 直ったか確認

ダウンロードが始まり、最後に「Installation complete」のような表示が出れば成功。新しいターミナルを開いて claude --version でチェック。

4. permission denied / EACCES

EACCES: permission denied, mkdir '/usr/local/lib/...' npm ERR! permission denied
💬 やさしく言うと

npm(ノードのアプリ管理ツール)で Claude Code を入れようとしたら、「そのフォルダに書き込む 許可がないよ」と言われている状態です。

⛔ 絶対やってはいけないこと

「許可がないなら管理者権限で…」と sudo npm install を使うのは NG。後々もっと大きなトラブルを引き起こします。

🔧 直し方(一番ラクな方法)

npm を諦めて ネイティブインストーラ に切り替えるのが圧倒的に簡単です。Node.js も不要になります。

  1. 新しいターミナルを開く
  2. 下のコマンドをコピーして貼り付け、Enter
curl -fsSL https://claude.ai/install.sh | bash
✅ 直ったか確認

ターミナルを開き直してから claude --version でバージョン番号が出れば成功。

5. ログインできない・ブラウザが開かない

Authentication failed ブラウザが開かない / ログイン画面に戻ってしまう
💬 やさしく言うと

Claude Code は、初回起動時にあなたが本人か確認するためにブラウザを開きます。それが何らかの理由で動いていない状態。

🔧 直し方(順番に)
  1. ターミナルに 長い URL が表示されているはず(https://claude.ai/... のようなもの)
  2. その URL を マウスで選択してコピー
  3. すでに claude.ai に ログイン済みのブラウザ を開く
  4. アドレスバーに貼り付けて Enter
  5. 「許可する」「Allow」のようなボタンを押す
  6. ターミナルに戻ると、自動で進んでいるはずです

それでもダメなら、Claude Code 内で /login と入力してやり直してください。

⚠️ あらかじめ確認

そもそも 有料プラン(Pro 以上) に入っていないとログインできません。claude.ai でプラン状況を確認してください。

6. 「Pro プランが必要」と言われる

This feature requires a Pro plan Your plan does not include Claude Code
💬 やさしく言うと

Web版 Claude(claude.ai)は無料でも使えますが、ターミナル版の Claude Code は無料プランでは使えません。Pro($20/月)以上が必要です。

🔧 直し方
  1. claude.ai/upgrade にアクセス
  2. 「Claude Pro」を選択
  3. 支払い情報を入力
  4. 登録できたら、ターミナルで Claude Code を再起動

最初は Pro で十分です。物足りなくなったら Max($100/月〜)を検討するくらいで OK。

7. 使いすぎでストップ(レート制限)

You have reached your usage limit rate_limit_exceeded
💬 やさしく言うと

「短時間に使いすぎたので、ちょっと休んでね」というメッセージ。Pro プランは 5時間ごとに使用枠がリセットされます。

🔧 直し方
  • すぐ使いたい場合: 数時間待つ。スマホでアラームをセットしておくと便利
  • 使用量を減らしたい: 大きな作業の前に /clear で履歴をリセット(過去の会話分も枠を消費しています)
  • 頻繁に当たる場合: Max プラン($100/月〜)に変更すると制限が大幅にゆるくなります
💡 普段から気をつけること

Claude は会話が長くなるほど消費が増えます。話題が変わったら /clear、別プロジェクトに移るときは exit して入り直すのがおすすめ。

8. Node.js のバージョンが古い(npm のとき)

npm WARN EBADENGINE Unsupported engine required: { node: '>=18.0.0' }
💬 やさしく言うと

npm でインストールしようとしたら、「Node.js が古すぎるよ。18 以上にして」と言われています。

🔧 直し方(一番ラクな方法)

npm を使わず、ネイティブインストーラ に切り替えれば Node.js 自体が不要になります。

curl -fsSL https://claude.ai/install.sh | bash
🛠 どうしても npm を使いたい場合
  1. nodejs.org から LTS版(緑のボタン)をダウンロード
  2. インストーラを実行
  3. ターミナルを完全に閉じて、開き直す
  4. node -v でバージョン確認(v20.x.x などが出ればOK)
  5. もう一度 npm install -g @anthropic-ai/claude-code

9. Claude がファイルを見つけてくれない

そのファイルは存在しません / I cannot find the file
💬 やさしく言うと

Claude Code は 「自分が起動された場所」のフォルダしか見ません。違うフォルダで起動していて、目的のファイルが見えていない状態です。

🔧 直し方
  1. Claude を一度終了する:
    → exit と入力して Enter、または Ctrl+C を 2回押す
  2. 正しいフォルダに移動する:
cd ~/Desktop/my-project

↑ my-project の部分は、自分のフォルダ名に置き換えてください。

  1. そこから claude を起動し直す
💡 今いる場所がわからなくなったら
  • Mac/Linux: pwd(今のフォルダのフルパスが表示される)
  • Windows: cd(同上)
  • ファイル一覧を見たい: ls(Mac/Linux)/ dir(Windows)

10. ターミナルでコピペができない

⌘+V / Ctrl+V を押しても何も起きない、貼り付けられない
💬 やさしく言うと

ターミナルの コピー&ペーストは、普通のアプリと操作が違うことがあります。慣れれば一発です。

🔧 OS別の正しい操作
  • Mac: コピー ⌘+C / 貼り付け ⌘+V(普通通り)
  • Windows Terminal / PowerShell:
    コピー Ctrl+Shift+C / 貼り付け Ctrl+Shift+V または 右クリック
  • Linux: 多くは Ctrl+Shift+C / Ctrl+Shift+V
💡 一番確実な方法

マウスで 右クリック してメニューから「貼り付け」を選ぶと、どの環境でも大体うまくいきます。

11. 動かなくなった・止めたい

Claude が長時間考え込んでいる / 暴走した気がする / 止めたい
💬 やさしく言うと

Claude が思った動きをしないとき、強制的に止める方法 があります。覚えておくと安心です。

🛑 強制ストップの方法
  1. Ctrl+C を 1回 押す → 今の動作をキャンセル
  2. それでも反応しなければ Ctrl+C を もう1回
  3. 完全に終了したいときは exit と入力して Enter
  4. 本当に固まったら、ターミナルウィンドウを ✕ で閉じてOK
⚠️ 安心ポイント

強制終了しても、パソコンが壊れることはありません。Claude Code は新しいターミナルを開けばまた起動できます。

12. 警告(WARN)が出ているけど大丈夫?

npm WARN deprecated ... warning: package xxx is outdated
💬 やさしく言うと

「WARN(警告)」と「ERROR(エラー)」は別モノです。

  • ERROR: 失敗した。何か対処が必要
  • WARN: 注意を促しているだけ。ほとんどの場合、無視してOK
✅ 確認方法

インストール完了後に claude --version でバージョンが表示されれば 成功しています。WARN がたくさん出ていても、最終的に動けば問題ありません。

13. 間違えて変なコマンドを実行した

変な記号が出た / 画面が真っ暗 / 入力できない
💬 やさしく言うと

たとえば vim や nano といったエディタが立ち上がってしまったり、引用符を閉じ忘れて入力受付状態になっていたりするケースです。

🆘 とりあえずの脱出コマンド(順に試す)
  1. Ctrl+C(キャンセル)
  2. Esc を 2〜3回押してから :q! と入力 Enter(vim から脱出)
  3. Ctrl+X → n → Enter(nano から脱出)
  4. それでもダメなら、ターミナルウィンドウを ✕ で閉じる
💡 予防策

知らないコマンドを実行するときは、まず Claude にきいて から実行する習慣をつけると安心です。

🆘 それでも解決しないとき

上のどれにも当てはまらなければ、以下を順に試してください。

  1. ターミナルで claude doctor を実行 → 出力をすべてマウスで選択してコピー
  2. 右下の 💬 ボタン から Claude を開く
  3. 「Claude Code でこんなエラーが出ています」とコピーした内容を貼り付けて質問
  4. それでも解決しなければ、公式 GitHub Issues で同じ問題が報告されていないか検索
  5. 最終手段は 公式サポート へ
💡 質問するときのコツ ① OS(Mac / Windows / Linux のどれか) ② 使ったコマンド(自分が打ったもの) ③ エラー文の全文。この3つを伝えれば、Claude も人間も的確に答えられます。
